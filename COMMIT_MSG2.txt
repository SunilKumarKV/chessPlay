fix: resolve Stockfish worker syntax error by bundling local WASM files

- Stockfish CDN URL (jsdelivr) was returning 404, causing blob to contain HTML error page
- Worker tried to parse HTML as JavaScript, causing 'SyntaxError: Invalid or unexpected token'

- Installed stockfish npm package (18.0.7)

- Updated useStockfish hook to fetch local files first, then CDN as fallback
- Added proper response status checking and content length verification
- Updated ESLint Flat Config to ignore public folder and build outputs
- Removed deprecated .eslintignore file

RESULT:
✓ Stockfish loads successfully from local bundled WASM files
✓ No more 'Invalid or unexpected token' syntax error
